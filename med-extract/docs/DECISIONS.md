# 决策记录

## 2025-12-25
- 建立符合 AGENTS 协议的文档框架（docs/ 与 status/ 目录）。
  - 背景：仓库仅有原始病历与现有技能目录，缺少任务、变更与日志记录，不利于多代理协作与追溯。
  - 决策：创建 `docs/ROADMAP.md`、`status/TASKS.md`、`status/CHANGELOG.md`、`status/JOURNAL.md`，并在后续工作中持续维护。
- 新建技能 `med-extract`，输出结构化摘要 + 时间轴 + 过错观察三段式格式，并通过引用 `references/extraction_schema.md` 统一字段。
  - 背景：用户需基于长篇病历快速了解诊疗过程并初步判断医疗过错。
  - 决策：选择可读性强的 Markdown 输出模板（案件概要/结构化字段/时间轴/过错线索），并在参考文件中固化字段定义，避免遗漏关键要素或生成臆测。
- 调整目录为独立项目树 `med-extract/`，将技能、docs、status 与 dist 打包文件集中管理。
  - 背景：仓库需承载多个法律技能项目，本技能应独立成树便于后续并行开发与 git 管理。
  - 决策：在独立目录下保留 `skill/` 供打包，`docs/`、`status/` 记录项目上下文，`dist/` 存放打包产物。
- 技能命名从 `medical-dispute-record-extractor` 精简为 `med-extract`。
  - 背景：原名过长不便触发和分发。
  - 决策：统一目录、包名与 frontmatter 为 `med-extract`，保持描述不变。
- 扁平化技能目录：`SKILL.md` 与 `references/` 置于 `med-extract/` 根，打包输出放置于仓库根的 `dist/`，避免嵌套目录和自包含压缩。
  - 背景：希望每个技能文件夹可直接复制使用，避免多余的 `skill/` 子目录；打包路径若置于技能内部会导致自包含膨胀。
  - 决策：移除 `skill/` 子目录；打包时使用 `python3 skills/skill-creator/scripts/package_skill.py med-extract dist`，确保 zip 只含技能与文档，不包含打包产物自身。
- 抽取策略转向“时间轴优先”，每个时间点必须记录诊断、手术/操作、术前判断、术后监测与风险告知。
  - 背景：律师更关注诊疗过程的时序与单点事件细节，便于判断过错与责任链条。
  - 决策：在 `med-extract/SKILL.md` 与 `med-extract/references/extraction_schema.md` 中用 Markdown 模板明确每条时间点的必填要素。
- 示例与样本全部匿名化，禁止保留真实客户病历材料。
  - 背景：技能包将被分发使用，包含真实客户信息存在隐私与合规风险。
  - 决策：移除真实病例文件，示例统一使用张三/李四等匿名占位符，并要求后续示例均脱敏。
- 规划新增“诊疗规范/过错评价/人损评定”资料模块作为前置筛查依据。
  - 背景：律师在前期阅卷阶段需要同步对照规范与鉴定标准，以便尽早识别过错与人损要点。
  - 决策：先在文档中建立需求与任务条目，待官方材料收集后再补充 references 与检索流程。
- 首批规范文件采用公开可下载的官方/期刊来源，未找到江苏省专项规范的公开 PDF。
  - 背景：用户要求“最新版本、中文、优先江苏省”，但公开检索范围内未发现江苏省专项文件。
  - 决策：先收录国家卫健委标准与权威期刊指南，并在 `guideline_checklist.md` 标注来源与替换说明，待用户提供或后续找到省级文件再更新。
