# Legal Skill Creator - 法律上下文智能提取器

本 Skill 是一个智能的价值发现工具，用于从律师事务所积累的历史文书中
自动识别和提取有价值的上下文，解决人工识别隐性知识的难题。

## 核心价值

- **专业技巧提取**：识别谈判策略、风险预警、证据组织、法律推理等技巧
- **写作风格学习**：学习资深律师的用词精准性、行文风格、语气把握
- **隐性知识显性化**：将经验丰富的律师的智慧转化为可复用的上下文
- **降低专业门槛**：让年轻律师快速掌握资深律师的专业技能

## 一、适用时机（When to use）

当出现以下需求时，应优先考虑调用本 Skill：

### 1. 技能与技巧提取
- 你已经有多份同类型的高质量法律文书，希望从中提取有价值的办案技巧：
  - 多份律师函 → 谈判策略与威慑技巧
  - 多份答辩状 → 抗辩思路与论证方法
  - 多份合同审查意见 → 风险识别模式
  - 多份客户沟通记录 → 沟通艺术与期望管理

### 2. 写作风格学习
- 希望学习某位资深律师的写作风格：
  - 用词特点与专业术语运用
  - 逻辑结构与表达方式
  - 不同场景的语气把握
  - 严谨性 vs 可读性的平衡

### 3. 创建新的技能模块
- 为某类法律场景创建可复用的 Skill family：
  - `ip_copyright_demand_letter` - 著作权侵权律师函技巧
  - `contract_review_risks` - 合同审查风险识别
  - `court_evidence_organization` - 庭审证据组织方法
  - `partner_writing_style` - 合伙人写作风格学习

## 二、向用户确认的信息

调用本 Skill 前，应主动向用户确认以下信息：

1. **技能家族标识（必填）**
   - 例如：`ip_copyright_demand_letter` / `ip_copyright_answer` / `ip_litigation_workflow`。

2. **法域与程序阶段（必填）**
   - 例如：`jurisdiction = CN`，`stage = pre_litigation / first_instance / second_instance / enforcement`。

3. **当事人侧别（必填）**
   - 例如：`party_role = rights_holder_counsel / defendant_counsel / licensee_counsel`。

4. **文书类型（必填）**
   - 例如：`demand_letter`、`complaint`、`answer`、`evidence_plan`、`workflow_memo` 等。

5. **语料来源（必填）**
   - 用户指定文件路径模式（如：`cases/copyright_letters/*.md`），
     或直接贴入若干已脱敏的文书内容；
   - 如文件有 YAML frontmatter，可从中读取 `doc_type`、`matter_type`、`jurisdiction`、
     `stage`、`quality` 等字段，用于过滤。

6. **质量过滤标准（选填）**
   - 如：仅使用标记为 `quality = gold` 的范本；
   - 或用户口头指定：“只用这几份我说好的版本”。

## 三、法律技能的定义框架

在提取技能之前，首先需要理解什么是法律技能。每个法律技能都可以通过以下七个维度来完整描述：

### 1. Context（上下文）
技能适用的场景、前提条件和环境背景。

**关键要素：**
- **法律领域**：如知识产权、合同法、劳动法等
- **案件类型**：如侵权纠纷、合同争议、劳动争议等
- **程序阶段**：如诉前、一审、二审、执行等
- **司法管辖区**：如中国大陆、香港、美国等
- **当事人角色**：如原告律师、被告律师、法律顾问等

### 2. Goal（目标）
技能要达成的具体目标或要解决的问题。

**关键要素：**
- **主要目标**：技能的核心目的
- **次要目标**：辅助达成的其他目标
- **成功标准**：判断目标达成的标准

### 3. Inputs（输入）
执行技能所需的信息、材料和数据。

**关键要素：**
- **必要信息**：执行技能必须具备的信息
- **可选信息**：有助于提高技能效果的信息
- **信息来源**：如何获取所需信息
- **信息质量**：对信息可靠性和完整性的要求

### 4. Workflow（工作流程）
执行技能的具体步骤和决策逻辑。

**关键要素：**
- **主要步骤**：执行技能的核心流程
- **决策点**：需要专业判断的关键节点
- **可选路径**：不同情况下的处理方式
- **质量控制**：确保质量的检查点

### 5. Outputs（输出）
技能执行的产物、结果和交付物。

**关键要素：**
- **主要产出**：技能执行的主要成果
- **附加产出**：过程中产生的其他有价值内容
- **格式要求**：产出的标准化格式
- **质量标准**：产出应该达到的质量水平

### 6. Constraints（约束）
法律限制、风险边界和注意事项。

**关键要素：**
- **法律限制**：必须遵守的法律规定
- **风险提示**：需要特别注意的风险点
- **道德约束**：职业道德和执业纪律要求
- **操作边界**：技能适用的范围和限制

### 7. Style（风格）
专业表达方式、语气和格式要求。

**关键要素：**
- **语气特点**：正式、严谨、威慑或协商
- **用词选择**：专业术语的准确使用
- **句式结构**：法律文书的典型句式
- **格式规范**：行业标准的格式要求

## 四、法律技能分类体系

基于法律技能的特性和应用场景，将法律技能分为六大类别：

### 1. 过程类技能
**定义：**管理案件进程、控制工作流程的技能
- 案件管理、流程控制、进度跟踪、截止日期管理

### 2. 文书类技能
**定义：**起草、审查、修改各类法律文书的技能
- 起诉状起草、合同起草、法律意见书、辩护词写作

### 3. 判断类技能
**定义：**进行法律分析、风险评估和策略选择的技能
- 法律适用分析、风险评估、证据判断、策略选择

### 4. 证据类技能
**定义：**收集、组织、呈现证据的技能
- 证据收集、证据组织、证据呈现、质证应对

### 5. 沟通类技能
**定义：**与客户、对方、裁判者等进行有效沟通的技能
- 客户沟通、谈判、庭审表达、协调沟通

### 6. 写作类技能
**定义：**运用法律语言进行专业表达的技能
- 逻辑构建、语言精炼、风格统一、专业化表达

## 五、内部操作步骤（Claude 内部遵循）

> 以下步骤为模型在执行 Legal Skill Creator 时的内部逻辑指引，无需逐条向用户复述。

### 第 1 阶段：收集与对齐语料

1. 使用 Glob 工具根据用户指定路径收集目标文件列表；
2. 使用 Read 读取所有目标文件内容；
3. 尝试解析每份文书的 YAML frontmatter（如存在），提取：
   - `doc_type`、`matter_type`、`jurisdiction`、`stage`、`party_role`、`quality` 等；
4. 根据用户提供的筛选条件和元数据，过滤掉明显不匹配或质量较低的文书。

### 第 2 阶段：智能价值发现

#### 2.1 基于七维度框架分析

对每份文书，使用七维度框架进行分析：
- 识别文书的上下文背景（Context）
- 明确文书要达成的目标（Goal）
- 列出所需的信息输入（Inputs）
- 梳理文书的工作流程（Workflow）
- 确定文书的输出要求（Outputs）
- 标记相关的约束条件（Constraints）
- 分析文书的表达风格（Style）

#### 2.2 模式识别与价值分析

深度分析文书内容，识别有价值的模式：

1. **专业技巧识别**：
   - 谈判策略：威慑、合作、妥协等策略的运用
   - 风险预警：如何识别和提示潜在风险
   - 证据组织：证据的呈现逻辑和说服技巧
   - 法律推理：从事实到结论的思维路径

2. **写作风格分析**（若关注风格）：
   - 用词特点：专业术语的使用频率和准确性
   - 句式结构：长短句搭配，逻辑连接词运用
   - 语气控制：不同场景下的语气转换
   - 表达习惯：常用表述和固定搭配

3. **价值评估**：
   - 独特性：是否是独特的或创新的解决方案
   - 普适性：是否适用于多个类似案件
   - 效果性：是否在过往案例中取得了良好效果
   - 可学性：是否容易被其他律师理解和应用

#### 2.3 关联分析

- 发现不同技巧之间的关联和组合模式
- 识别可组合的技巧套件
- 构建从通用到专业的知识体系

### 第 3 阶段：理解使用场景与规划资源

基于价值发现的结果，完成以下工作：

#### 3.1 理解使用场景
1. **分析使用模式**：
   - 技能将被如何使用？（如：起草律师函、审查合同、组织证据）
   - 典型的用户请求是什么？
   - 什么情况下会触发这个技能？

2. **识别触发条件**：
   - 明确什么情况下应该使用这个技能
   - 描述技能的适用边界和限制

#### 3.2 规划可复用资源
1. **Scripts 资源**（需要确定性执行或反复重写的代码）：
   - 是否需要固定的脚本？（如：风险评估计算器、条款生成器）
   - 是否有需要反复执行的操作？

2. **References 资源**（需要参考的文档）：
   - 是否有法律条文模板需要参考？
   - 是否有风险清单需要查阅？
   - 是否有案例库需要引用？

3. **Assets 资源**（输出中使用的文件）：
   - 是否需要文书模板？
   - 是否需要检查清单？
   - 是否有示例文件？

### 第 4 阶段：生成 Claude Skill

基于以上分析，直接生成符合 Claude Code 规范的 Skill：

#### 4.1 创建 SKILL.md
1. **YAML frontmatter**：
   - `name`: 简短、描述性的技能名称
   - `description`: 清晰说明技能的功能和使用场景

2. **技能描述**：
   - 技能的目的和价值
   - 适用时机和触发条件
   - 使用方式和步骤

3. **核心工作流程**：
   - 基于"智能价值发现"的结果
   - 描述具体的工作步骤
   - 说明如何使用规划的资源

4. **输出格式**：
   - 明确说明将产生什么
   - 提供输出结构或模板

5. **安全提示**：
   - 法律责任声明
   - 风险提示和注意事项

#### 4.2 创建配套资源（如需要）
1. **创建 scripts/ 目录**（如有需要）
2. **创建 references/ 目录**（存放参考文档）
3. **创建 assets/ 目录**（存放模板和示例）

### 第 5 阶段：与用户确认并完善

1. **展示生成结果**：
   - 技能目录路径和名称
   - SKILL.md 的主要内容概要
   - 配套资源说明（如有）

2. **收集反馈**：
   - 技能是否符合预期
   - 是否需要调整工作流程
   - 是否需要补充资源

3. **迭代优化**：
   - 根据反馈修改 SKILL.md
   - 调整配套资源
   - 完善使用说明

### 第 6 阶段：包装与交付

1. **验证技能**：
   - 检查 SKILL.md 格式是否符合 Claude Code 规范
   - 确认资源文件组织合理
   - 验证技能描述清晰准确

2. **创建使用指南**：
   - 说明如何调用该技能
   - 提供典型使用示例
   - 列出注意事项和限制

## 六、输出内容

本 Skill 的输出应至少包括：

1. **生成的 Skill 目录**：
   - 包含完整的 SKILL.md 文件
   - 配套资源文件（scripts/、references/、assets/）
   - 目录路径和访问方式

2. **技能摘要说明**：
   - 技能的核心功能和价值
   - 适用的场景和条件
   - 主要工作流程概述

3. **使用指南**：
   - 如何在 Claude Code 中调用该技能
   - 典型的使用示例
   - 注意事项和限制条件

## 七、核心原则

### 领域边界清晰
- 仅在用户明确提供法域与程序阶段后再进行抽象
- 对于不确定或超出语料支持范围的推断，宁可保守，不要臆测

### 以多份文书的共同模式为主
- 不要因一份特殊文书而让技能定义过度偏向"极端个案"
- 优先寻找多个范本中共现的字段、步骤与结构块

### 尊重律师的真实办案习惯
- 遇到存在多种合理路径的情况，应标注为「可选变体」而非强行统一
- 重要的风格约束和风险提示要显式写入

### 保守的风险处理
- 避免在证据不足时作绝对性事实认定
- 不作关于诉讼结果或赔偿数额的保证性承诺
- 保持专业、克制的语气，避免不当威胁

## 八、安全与合规

- 本 Skill 仅用于价值发现与技能抽象，不直接提供具体案件的最终法律意见
- 所有生成的 Skill 必须明确提示：
  - 需要由具备相应执业资质的律师进行最终审核
  - 不构成对具体案件结果的承诺或保证
  - 在证据不足的情形下应避免使用绝对性措辞
- 不应根据不完整或质量明显较差的文书做过度概括
